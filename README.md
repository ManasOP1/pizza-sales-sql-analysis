# pizza-sales-sql-analysis
This project analyzes a pizza restaurant's sales data using SQL to uncover business insights related to revenue, customer purchasing behavior, product performance, and sales trends.

The objective was to answer key business questions that could help restaurant management make data-driven decisions regarding inventory planning, menu optimization, and revenue growth.

🛠️ Tools Used
SQL (MySQL)
Relational Database Concepts
Joins
Aggregate Functions
Subqueries
Common Table Expressions (CTEs)
Window Functions
📂 Dataset Information

The dataset consists of four tables:

Orders

Contains order date and time information.

Order Details

Contains pizza quantities ordered in each transaction.

Pizzas

Contains pizza sizes and pricing information.

Pizza Types

Contains pizza names and categories.

🎯 Business Problems Solved
Basic Analysis

✅ Calculated the total number of orders placed.

✅ Calculated total revenue generated from pizza sales.

✅ Identified the highest-priced pizza.

✅ Determined the most commonly ordered pizza size.

✅ Identified the top 5 most ordered pizzas based on quantity sold.

Intermediate Analysis

✅ Calculated total quantity sold for each pizza category.

✅ Analyzed order distribution by hour of the day.

✅ Determined category-wise distribution of available pizzas.

✅ Calculated average number of pizzas ordered per day.

✅ Identified the top 3 pizzas generating the highest revenue.

Advanced Analysis

✅ Calculated the percentage contribution of each pizza category to total revenue.

✅ Analyzed cumulative revenue growth over time using window functions.

✅ Identified the top 3 revenue-generating pizzas within each category using ranking functions.
